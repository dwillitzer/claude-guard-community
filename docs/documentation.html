Claude Guard Community Edition Documentation

Installation

Claude Guard requires Node.js 14.16 or higher and the Claude CLI to be installed first.

# 1. Install Claude CLI first
npm install -g @anthropic-ai/claude-code

# 2. Install Claude Guard
npm install -g claude-guard

# 3. Verify installation
claude-guard --version

Basic Usage

Claude Guard is a drop-in replacement for the claude command with added security features.

# Use exactly like Claude CLI
claude-guard "explain this error message"

# Debug assistance
claude-guard "help me optimize this function"

# Code generation
claude-guard "create a REST API endpoint for user auth"

Claude Guard now supports Claude's native .claude/settings.json format with hybrid validation.

Setting Up Claude Settings

Create a .claude/settings.json file in your project root:

{
  "permissions": {
    "allow": [
      "Bash(git *)",
      "Bash(npm *)",
      "Bash(ls*)",
      "Read(*)",
      "Edit(*)",
      "Write(**)"
    ],
    "deny": [
      "Bash(rm -rf /*)",
      "Bash(sudo rm -rf /*)",
      "Bash(dd if=/dev/zero of=/dev/*)",
      "Bash(shutdown*)",
      "Bash(reboot*)"
    ]
  }
}
‚úÖ Benefits:
  • Standards Compliance: Uses Claude's official settings format
  • Layered Security: Claude rules + additional guard patterns
  • Easy Migration: Existing .claude/settings.json files work immediately
  • Backwards Compatible: Guard-only mode preserved

Enabling Integration

Enable Claude settings integration in your guard configuration:

# ~/.claude/guard/config.json
{
  "policies": {
    "useClaudeSettings": true,
    "claudeSettingsFirst": true
  }
}

Configuration

Claude Guard can be configured through ~/.claude/guard/config.json:

{
  "version": "2.1",
  "policies": {
    "useClaudeSettings": true,
    "claudeSettingsFirst": true,
    "blockedPaths": ["/etc", "/usr/bin/rm", "/usr/bin/dd"],
    "allowedCommands": ["*"],
    "blockedCommands": [
      "rm -rf /",
      "dd if=/dev/zero of=/dev/*",
      "sudo rm -rf /",
      "shutdown -h now",
      "reboot"
    ],
    "warnCommands": [
      "rm -rf *",
      "sudo *",
      "chmod 777 *"
    ],
    "maxCommandLength": 1000,
    "allowShellExpansion": false,
    "requireConfirmation": ["rm -rf *", "sudo *"]
  },
  "aliases": {
    "@test": "run all tests in this project",
    "@build": "build the project for production",
    "@deploy": "deploy to production server"
  }
}

Security Features

Validation Flow

Claude Guard uses a hybrid validation system:

Command ‚Üí Claude deny patterns ‚Üí ‚ùå BLOCKED
       ‚Üí Claude allow patterns ‚Üí ‚úÖ ALLOWED  
       ‚Üí Guard patterns ‚Üí ‚ùå BLOCKED or ‚úÖ ALLOWED

Default Protections

üö´ Blocked Actions

  • System destruction: rm -rf /
  • Credential access: SSH keys, passwords
  • Network attacks: Port scanning
  • Privilege escalation: Unauthorized sudo

‚úÖ Allowed Actions

  • Development tools: npm, yarn, pip
  • Version control: git operations
  • File operations: reading, listing
  • Safe utilities: curl to known domains
